Java'da aritmetik operatörler konumuza devam ediyoruz. Bir önceki slaytlarda işin genel kurallarını ve bazı temel tuzakları konuşmuştuk. Ancak bu slayt, Java'nın arka planında (under the hood) dönen o gizli ve ince detaylara odaklanıyor. Yazılımcıların kod yazarken en çok "Bu hata nereden çıktı?" dediği, mülakatçıların ise adayları köşeye sıkıştırmak için bayıldığı üç can alıcı konuyu inceleyeceğiz: Sıfıra bölünme tepkileri, karakterlerin sihirli bir şekilde sayıya dönüşmesi ve küçük veri tiplerinin işlemlerdeki gizli terfisi.
Hemen sol taraftaki teorik kurallarımızla başlayalım. İlk konumuz Sıfıra Bölünme (Zero Division). Klasik matematikte bir sayıyı sıfıra bölemezsiniz, Java da bunu bilir. Ancak burada çok büyük bir ayrım var: Eğer iki tam sayıyı (int) sıfıra bölmeye çalışırsanız, Java programı anında durdurur ve ArithmeticException hatası fırlatarak çöker. Fakat iş ondalıklı sayılara (double) geldiğinde Java çökmez! Bunun yerine bize Infinity (Sonsuz) veya NaN (Not a Number - Sayı Değil) gibi özel matematiksel sonuçlar verir.
İkinci detayımız Karakter (Char) Matematiği. Java'da char veri tipleri (örneğin 'A' harfi) aslında arka planda birer sayıdır; yani ASCII veya Unicode karşılıklarını tutarlar. Eğer iki karakteri veya bir karakterle bir sayıyı toplamaya çalışırsanız, Java bunları yan yana yapıştırmaz. Karakterin arka plandaki sayısal değerini (örneğin 'A' için 65) alır ve matematiksel olarak toplar.
Üçüncü ve en gizli tuzak ise Gizli Tip Terfisi (Promotion). Java'da byte veya short gibi küçük veri tipleri üzerinde aritmetik bir işlem yaptığınızda, Java bu değişkenleri işlem sırasında otomatik olarak int tipine yükseltir (terfi ettirir). Bu yüzden iki byte sayıyı toplayıp doğrudan başka bir byte değişkene atayamazsınız; Java size derleme hatası fırlatır.
Bu durumların bellek seviyesinde ve işlem akışında nasıl gerçekleştiğini orta bölümdeki akış şemamızla somutlaştıralım. İlk olarak sıfıra bölünme senaryolarına bakalım: 5 / 0 işlemi tam sayı bölmesi olduğu için anında "HATA! (Crash)" durumuna yol açar. Ancak bunu 5.0 / 0 şeklinde ondalıklı yazarsanız sonuç Infinity olur. Eğer 0.0 / 0 gibi belirsiz bir bölme yaparsanız da sonuç NaN olur.
Hemen altında karakter işlem akışını görüyoruz: 'A' + 1 yazdığımızda, Java 'A' karakterinin ASCII değeri olan 65'i alır, üzerine 1 ekler ve sonucu 66 olarak üretir.
En altta ise byte toplama tuzağının görselini görüyoruz: byte x = 10 ve byte y = 20 olarak tanımlanmış. Biz byte z = x + y yazmaya çalıştığımız anda Java buna "HATA!" der. Çünkü arka plandaki işlem int olarak gerçekleşmiştir ve büyük bir veri tipi (int), daha küçük bir kutuya (byte) doğrudan sığdırılamaz.
Gelin bu ince detayların kod içinde nasıl can bulduğunu en sağdaki kod bloğunda inceleyelim. Birinci bölümde sıfıra bölünme testlerini görüyoruz. int hata = 5 / 0; satırı yorum satırına alınmış çünkü eğer aktif olsaydı programımızı çökertecekti. Hemen altında double sonsuz = 5.0 / 0; yazarak Infinity sonucunu, double belirsiz = 0.0 / 0.0; yazarak da NaN sonucunu güvenle elde ediyoruz.
İkinci bölüme geçelim: Karakter Matematiği. char harf = 'A'; olarak tanımlanmış. Biz int charToplami = harf + 5; yazdığımızda, harf değişkeninin ASCII değeri olan 65 ile 5 toplanıyor ve charToplami değişkenimize 70 değeri atanıyor. Eğer bir sonraki harfi elde etmek isteseydik, char siradakiHarf = (char) (harf + 1); yazarak sonucu (char) 66 yani 'B' olarak elde edebilirdik; burada manuel bir dönüştürme (casting) yapmamız gerektiğini unutmayın.
Son bölüm ise o meşhur Gizli Tip Terfisi tuzağını kanıtlıyor. byte b1 = 10; ve byte b2 = 20; değişkenlerimiz var. byte b3 = b1 + b2; satırı yorum satırında çünkü bu satır derleme hatası verir. Doğru yöntem, sonucu int olarak karşılamak ve int toplam = b1 + b2; şeklinde yazmaktır. Java'nin bu arka plan kurallarını ve bellek yönetimini iyi bilmek, sizi beklenmedik çökmelerden ve derleme hatalarından tamamen koruyacaktır.
