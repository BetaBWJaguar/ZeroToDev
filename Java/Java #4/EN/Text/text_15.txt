Now that we have fully mastered arithmetic operators, we are moving on to the most fundamental and frequently used operator family in Java: Assignment Operators. At first glance, you might think, "The equals sign just assigns a value, how deep can the logic really be?" However, assignment in Java works completely differently from standard mathematics. In math, the equals sign indicates that both sides are balanced, but in Java, it acts like a one-way street; it takes the evaluated values on the right and places them inside the variables on the left. And the golden rule of this operation is that execution flows strictly from Right to Left. As we specified in our slide, Java provides us with both this basic assignment and compound assignment operators that simplify our code.
Let's start with our theoretical panel on the left. Our first topic is Basic Assignment (=). This operator takes the value on its right side and copies it into the variable on its left. The critical rule here is that the left side can never be a value or a mathematical operation; it must be a variable, and only a variable. For example, you cannot write 5 = x; as this would cause a compiler error. Another practical feature is Chaining. You can assign the same value to multiple variables in a single line; for example, when you write a = b = 10;, Java runs perfectly.
Our second topic is Compound Assignment operators, which make our code more professional. We already know that the expression x += 5 is the exact shorthand for the long form x = x + 5. However, there is a major hidden magic—a Pro-Tip—that you will often encounter in interviews: Implicit Casting. Under normal circumstances, if you try to add an int value (which 5 is by default) to a variable of a smaller type like byte, Java will throw a compiler error. But when you do this using a compound operator (+=), Java automatically handles this type conversion behind the scenes and runs without any errors.
Let’s visualize how this right-to-left execution mechanism happens in memory and during operation using our flowchart in the middle section. First, let's look at standard assignment: we write int x = 10 + 5;. Java does not read from left to right; it sees the right side first. In Step 1, it calculates the right side 10 + 5 and gets 15. Only in Step 2 does it transfer this value of 15 into the x variable on the left.
Now let's compare this with compound assignment: we write x += 5;. In Step 1, Java expands this in memory as x = x + 5. In Step 2, it calculates the mathematical operation on the right side 15 + 5 and gets 20. In Step 3, it writes this final value of 20 back into the x variable on the left, updating the old value.
Now, let's examine how all these rules run smoothly in our code block on the right. In the first section, we see basic and chained assignment: we declare two variables with int a, b;. When we write a = b = 10;, because of the right-to-left execution rule, Java first assigns 10 to b, and then copies this value of 10 from b into a. Thus, we make both variables 10 in a single line.
In the second section, we run our compound assignment operator. We start with int skor = 50;. With skor += 10;, our score becomes 60; it falls to 40 with skor -= 20;, and finally we find our ultimate score as 20 with skor /= 2;.
The third section proves that hidden conversion magic I just mentioned. We define a small data type like byte kucukSayi = 5;. The line kucukSayi = kucukSayi + 5; is commented out because it would throw a compiler error. However, the line right below it, kucukSayi += 5;, runs perfectly and makes the result 10; because the += operator silently handles the automatic type conversion. When we print the outputs at the end of the program, we see that the values of A and B are 10, the Son Skor (Final Score) is 20, and our Byte Değer (Byte Value) is 10. Since you have put the right-to-left execution logic of assignment operators and this hidden conversion magic into your pocket, you are now much more powerful in variable management in Java!
