In Java, variables act as named memory locations that are essential for storing, modifying, 
and managing data during the execution of a program. Since Java requires strict data management, 
every variable must be explicitly declared with a specific data type before it can be used. 
This data type requirement is critical because it dynamically determines both the amount of 
memory allocated and the exact kind of value the variable can hold. For instance, the int type is used for storing whole numbers, double 
for decimal values, and String for handling textual data. By utilizing variables in this structured manner, programs gain the necessary 
flexibility to process different types of information seamlessly.

The lifecycle of a variable within a program follows a very clear and logical 
progression. When the execution starts, the first step is to declare the variable 
by specifying its data type and assigning it a unique name. Following this declaration, a 
specific value is assigned to the variable, making it ready for active use in various operations, 
such as mathematical calculations or screen outputs. In a practical code structure, once variables 

like int age = 21; or String name = "Java"; are declared and initialized, their stored values 
can be easily displayed using standard commands like System.out.println(). This systematic 
flow—from declaration to assignment and usage—ensures that data is handled in an organized and error-free way.
