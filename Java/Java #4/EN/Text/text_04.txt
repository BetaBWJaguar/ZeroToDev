In Java, date and time information is stored by default in a standard and technical format that computers can easily process. However, when this data needs to be presented 
to the end user (for example, on an invoice, a ticket, or an interface), we need the readable and aesthetic appearances that everyone is accustomed to. 
This is where the DateTimeFormatter class comes into play. This class is the most fundamental tool used to convert the raw date and time objects we hold in our programs 
into user-friendly, flexible, and understandable text expressions. Increasing the readability of date-time output is one of the most important steps for the user experience of a
professional application.


The biggest advantage offered by DateTimeFormatter is the freedom it gives developers to create custom format patterns. By using symbols like "dd" for the day, "MMMM" to 
specify the month name in text, "yyyy" for the year, and "HH:mm" for hour-minute, outputs can be obtained in limitless combinations. This flexible structure works in 
perfect harmony with the modern time classes we discussed earlier, such as LocalDateTime, LocalDate, or ZonedDateTime. Another critical capability of the class is its 
support for "Locale-based formatting". Thanks to this feature, it becomes possible to format the exact same date data according to completely different languages and 
regional standards; for instance, month names are automatically converted to text according to the selected country's language.


The workflow of this system within a program follows a highly logical sequence. When the application needs to display date data, a raw date-time object is 
first created. When examining the code example on the slide, it is clearly seen how powerful this process is: the current time 
is retrieved from the system with LocalDateTime.now(). Then, two different DateTimeFormatters are defined. The first is prepared with the "dd MMMM yyyy HH:mm" 
pattern and Turkish Locale information in accordance with Turkish standards; the second is prepared with the "MMMM dd, yyyy hh:mm a" pattern (with AM/PM notation) 
and US Locale information in accordance with American standards. The formatter objects process this raw data to produce readable texts, and in the final stage, the 
results are printed to the screen using System.out.println() commands. This flow, extending from definition to formatting, ensures that a single system clock is presented 
to users in different parts of the world in their own languages and the formats they are used to.
