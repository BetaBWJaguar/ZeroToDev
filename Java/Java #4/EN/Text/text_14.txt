First, let's take a quick look at our Golden Rules to Remember on the left. In Java, we have our "Big 5" that make our mathematical universe spin: addition, subtraction, multiplication, division, and the modulus (%) operator used to find remainders. When performing calculations, we must always respect operator precedence; multiplication, division, and modulo always run before addition and subtraction. If we want to change this order, we use the power of parentheses (()) to override all rules. We must also never forget our other golden rule, integer division: when we divide two integers, Java discards the decimals, so if we want precision, at least one of our numbers must be written as a double. Finally, to make our code cleaner and more readable, we should make shorthand operators like ++ or += a regular habit.
So, how do we turn these rules into practical instincts when writing code? The Developer Checklist in the middle column guides us exactly on this. When coding, ask yourself these questions: "Am I checking for odd or even numbers?" If the answer is yes, the modulus % 2 should instantly come to mind. "Do I want decimal precision in a division operation?" Then write one of the numbers in a decimal format, like 2.0 or 4.0. "Do I want to determine the order of operations myself?" Immediately wrap that mathematical expression in parentheses (). "Am I going to update a variable's value by adding to itself?" Don't waste time writing the long form x = x + 5, use the x += 5 shortcut directly.
