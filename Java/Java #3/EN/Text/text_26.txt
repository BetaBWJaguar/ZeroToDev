When working with the Scanner class in Java, a common issue appears when methods like nextInt() and nextLine() are used one after another. The root of the problem is how Scanner manages the input buffer. When a user enters a value, the entire input (for example, 25\n) is placed into the buffer. The nextInt() method reads only the integer part (25) and stops when it reaches the newline character, but it does not remove that newline from the buffer.

Because of this, when nextLine() is called immediately after nextInt(), it finds the leftover newline character already waiting in the buffer. Since nextLine() reads until it encounters a newline, it consumes that character instantly and returns an empty string instead of waiting for user input. This makes it look like the program skipped the input step.

To handle this correctly, you need to manually clear the buffer after using nextInt(). This is typically done by adding an extra nextLine() call right after nextInt() to consume the remaining newline. After the buffer is cleared, the next nextLine() call will behave normally and wait for the user to enter a full line of text. This small adjustment ensures that different Scanner methods work together without causing unexpected behavior.
